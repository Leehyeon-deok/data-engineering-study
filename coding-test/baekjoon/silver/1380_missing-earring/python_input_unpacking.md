# Python 입력 언패킹 & 카운팅 패턴 정리

코딩 테스트와 알고리즘 문제에서 자주 등장하는 **입력 언패킹(unpacking)** 과 **카운팅 배열(counting array)** 패턴을 정리한다. 본 문서는 실전 문제에서 바로 써먹을 수 있도록 *의도, 문법, 예제, 주의점* 위주로 설명한다.

---

## 1. 입력 분리: `input().split()`

### 개념

* `input()` : 한 줄을 문자열로 입력받음
* `split()` : 공백 기준으로 문자열을 분리하여 **리스트**로 반환

### 예시

```python
s = input()          # "1 B"
parts = s.split()    # ["1", "B"]
```

---

## 2. 언패킹(Unpacking)

### 기본 형태

```python
a, b = ["1", "B"]
```

* 오른쪽의 값들이 **순서대로** 왼쪽 변수에 할당됨
* 개수가 맞지 않으면 에러 발생

### 입력과 함께 쓰기

```python
num, flag = input().split()
```

---

## 3. 버릴 값은 `_` 로 받기 (관용 표현)

### 의미

* `_` 는 **"이 값은 사용하지 않는다"** 는 의도를 표현하는 변수명
* 파이썬 문법 키워드는 아니지만, 업계 표준 관용

### 예시 (두 번째 값 무시)

```python
num, _ = input().split()
```

* `num` : 사용
* `_` : 의도적으로 무시

### 장점

* 가독성 ↑
* 코드 의도 명확
* 코딩 테스트에서 매우 선호됨

---

## 4. 숫자 변환과 연결

입력으로 받은 값은 **문자열**이므로 숫자 연산이나 인덱스로 쓰려면 변환이 필요하다.

```python
num = int(num)
```

---

## 5. 카운팅 배열 (Counting Array)

### 목적

* 특정 번호(또는 값)의 **등장 횟수**를 저장

### 선언

```python
count = [0] * (n + 1)
```

### 왜 `n + 1` 인가?

* 번호가 `1 ~ n` 범위
* 리스트 인덱스를 번호와 **1:1 대응**시키기 위함
* `count[0]` 은 사용하지 않음

### 구조 예시 (n = 3)

```
index:  0  1  2  3
count: [0, 0, 0, 0]
```

---

## 6. 카운팅 사용 예

```python
num, _ = input().split()
num = int(num)
count[num] += 1
```

* 입력이 등장할 때마다 해당 번호의 횟수 증가
* 결과적으로

  * `2` → 정상
  * `1` → 누락 / 분실 / 미완료 상태

---

## 7. 반복문에서 `_` 사용

### 사용하지 않는 반복 변수

```python
for _ in range(5):
    print("hello")
```

* 반복 횟수만 필요하고
* 인덱스 값은 필요 없는 경우

---

## 8. 자주 쓰이는 패턴 요약

### 입력 언패킹 + 카운팅

```python
count = [0] * (n + 1)

for _ in range(k):
    num, _ = input().split()
    count[int(num)] += 1
```

---

## 9. 주의사항

* 언패킹 시 **변수 개수와 값 개수는 반드시 일치**해야 함
* `_` 에 저장된 값은 다시 사용하지 않는 것이 관례
* `n+1` 배열을 쓰면 인덱스 실수를 크게 줄일 수 있음

---

## 10. 한 줄 요약

> `num, _ = input().split()` 은 **입력 중 필요한 값만 명확히 사용하겠다는 선언**이며,
> `count = [0] * (n + 1)` 은 **번호를 그대로 인덱스로 쓰기 위한 카운팅 배열 패턴**이다.
