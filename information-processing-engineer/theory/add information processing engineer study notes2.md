# 📚 정보처리기사 실기 핵심 정리

---

# 1. 리팩토링(Refactoring)

## 정의

기능은 변경하지 않고 **내부 구조를 개선하는 작업**

## 목적

- 복잡한 코드 단순화
- 소스 코드 가독성 향상
- 유지보수성 향상
- 유연한 시스템 구축
- 개발 생산성 향상
- 소프트웨어 품질 향상

### 암기

> **기능은 그대로, 구조만 개선**

---

# 2. 형상 통제(Configuration Control)

## 정의

형상 항목의 **버전 관리**를 위해 변경 여부와 변경 활동을 통제하는 활동

### 핵심 키워드

- 변경 관리
- 버전 관리
- 변경 승인

### 암기

> **형상 통제 = 변경 + 버전 관리**

---

# 3. 형상 관리 절차

## ① 형상 식별(Configuration Identification)

- 형상 관리 대상 정의
- 형상 항목 식별

### 암기

> **무엇을 관리할지 결정**

---

## ② 형상 통제(Configuration Control)

- 변경 요청 관리
- 변경 승인
- 형상통제위원회(CCB) 운영

### 암기

> **변경을 승인하고 관리**

---

## ③ 형상 감사(Configuration Audit)

- 변경 사항이 올바르게 반영되었는지 검증
- 소프트웨어 베이스라인 무결성 평가

### 암기

> **제대로 변경되었는지 검사**

---

## ④ 형상 기록(Configuration Status Accounting)

- 변경 이력 기록
- 형상 상태 관리
- 수행 결과 문서화

### 암기

> **변경 내용을 기록**

---

## 형상 관리 절차 암기

```
식 → 통 → 감 → 기
```

```
식별 → 통제 → 감사 → 기록
```

---

# 4. 헝가리안 표기법(Hungarian Notation)

## 정의

변수명 **앞(접두어)에 자료형이나 용도를 붙여** 작성하는 명명 규칙

### 예시

```java
int iCount;
String strName;
boolean bFlag;
char chGrade;
```

### 자주 사용하는 접두어

| 접두어 | 의미 |
|--------|------|
| i | int |
| str | String |
| b | boolean |
| ch | char |
| f | float |
| dbl | double |
| arr | 배열(Array) |
| obj | 객체(Object) |

### 암기

> **접두어 = 자료형 표시**
