# 📚 정보처리기사 핵심 정리

---

# 1. 테스트 종류

| 테스트 | 설명 |
|---------|------|
| **단위 테스트(Unit Test)** | 개별 모듈(함수, 클래스)이 정상 동작하는지 확인 |
| **통합 테스트(Integration Test)** | 모듈 간 인터페이스가 정상 동작하는지 확인 |
| **시스템 테스트(System Test)** | 시스템 전체가 요구사항을 만족하는지 확인 |
| **인수 테스트(Acceptance Test)** | 고객(사용자)이 최종 승인하는 테스트 |
| **알파 테스트(Alpha Test)** | 개발 환경에서 일부 사용자가 수행하는 테스트 |
| **회귀 테스트(Regression Test)** | 수정 후 기존 기능이 정상 동작하는지 재검사 |

### 💡 암기법
- 단 = 단일 모듈
- 통 = 통신(인터페이스)
- 시 = 시스템 전체
- 인 = 인수(고객 승인)
- 알 = 개발사 내부
- 회 = 수정 후 재테스트

---

# 2. IPC (Inter Process Communication)

### 정의
프로세스 간 데이터를 주고받기 위한 통신 기술

### 종류
- 공유 메모리 (Shared Memory)
- 소켓 (Socket)
- 세마포어 (Semaphore)
- 메시지 큐 (Message Queue)
- 파이프 (Pipe)

### 💡 암기법
> **프로세스끼리 대화 = IPC**

---

# 3. 데이터 모델 구성 요소

| 구성 요소 | 설명 |
|-----------|------|
| **구조(Structure)** | 데이터를 어떻게 저장할 것인가 |
| **연산(Operation)** | 데이터를 어떻게 처리할 것인가 |
| **제약조건(Constraint)** | 데이터 무결성을 유지하기 위한 규칙 |

### 💡 암기법
> **구 · 연 · 제**

- 구조 → 저장
- 연산 → 처리
- 제약조건 → 규칙

---

# 4. 결합도(Coupling)

## 결합도 순서 (강 → 약)

> **내용 > 공통 > 외부 > 제어 > 스탬프 > 자료**

➡️ 왼쪽일수록 결합도가 강하고 유지보수가 어렵다.  
➡️ 오른쪽일수록 결합도가 약하고 유지보수가 쉽다.

---

## 내용 결합도 (Content Coupling)

### 정의
다른 모듈의 내부 구현이나 데이터를 직접 참조하거나 수정하는 경우

### 특징
- 결합도 **가장 높음**
- 유지보수 **가장 어려움**

### 💡 암기법
> **남의 집에 들어가 가구까지 바꾸는 것**

➡️ **내용까지 간섭 = 최악**

---

## 자료 결합도 (Data Coupling)

### 정의
필요한 데이터만 매개변수로 전달하는 경우

### 특징
- 결합도 **가장 낮음**
- 유지보수 **가장 쉬움**

### 💡 암기법
> **택배만 전달하는 것**

➡️ **자료만 전달 = 최고**

---

# ⭐ 시험 암기 포인트

- 단위 테스트 = 개별 모듈
- 통합 테스트 = 인터페이스
- 시스템 테스트 = 전체 시스템
- 인수 테스트 = 고객 승인
- 회귀 테스트 = 수정 후 재검사
- IPC = 프로세스 간 통신
- 데이터 모델 = 구조 · 연산 · 제약조건
- 결합도 순서 = **내용 > 공통 > 외부 > 제어 > 스탬프 > 자료**
- 내용 결합도 = **남의 집 가구까지 바꿈(최악)**
- 자료 결합도 = **택배만 전달(최고)**
