# 1. Join(조인, ⋈)

두 개 이상의 테이블을 **공통 속성(키)** 을 이용하여 하나의 결과로 합치는 연산이다.

## 관계대수 기호

```
⋈
```

## SQL

```sql
SELECT *
FROM Student S
JOIN Score SC
ON S.StudentID = SC.StudentID;
```

## 종류

### INNER JOIN

양쪽 테이블에 모두 존재하는 데이터만 조회

### LEFT OUTER JOIN

왼쪽 테이블은 모두 조회

오른쪽에 없으면 NULL

### RIGHT OUTER JOIN

오른쪽 테이블은 모두 조회

왼쪽에 없으면 NULL

### FULL OUTER JOIN

양쪽 테이블의 모든 데이터를 조회

없는 값은 NULL

### CROSS JOIN

모든 경우의 수(Cartesian Product)

### SELF JOIN

자기 자신과 조인

# 2. FLSM (Fixed Length Subnet Mask)

모든 서브넷의 크기를 동일하게 나누는 방식이다.

예)

```
192.168.1.0/24
```

4개의 서브넷으로 분할

```
/26
```

생성

```
192.168.1.0

192.168.1.64

192.168.1.128

192.168.1.192
```

## 특징

- 모든 서브넷 크기가 동일
- 관리가 쉬움
- IP 주소 낭비 가능

### 핵심

비트를 n개 빌리면

```
2ⁿ
```

개의 서브넷이 생성된다.

예)

```
1비트 빌림

↓

2¹

↓

2개
```

```
2비트 빌림

↓

2²

↓

4개
```

```
3비트 빌림

↓

2³

↓

8개
```

즉,

FLSM은 비트를 빌려 서브넷을 만들기 때문에

항상

```
1개

2개

4개

8개

16개

32개
```

처럼 **2의 제곱 개수**만 생성된다.

# 3. SIEM (Security Information and Event Management)

보안 정보 및 이벤트 관리 시스템이다.

다양한 보안 장비에서 발생하는 로그를 **수집·분석·모니터링**하여 보안 위협을 탐지하고 대응한다.

## 특징

- 다양한 보안 장비의 로그를 중앙에서 통합 관리
- 로그의 상관관계 분석
- 이상 행위 탐지
- 실시간 모니터링
- 이벤트 발생 시 알림
- 머신러닝 기반 이상 패턴 분석 가능

## 주요 기능

- 로그 통합 수집
- 이벤트 분석
- 행위 기반 분석
- 문맥(Context) 기반 분석
- 이상 패턴 탐지
- 위협 대응

### 핵심 암기

SIEM

↓

로그 수집

↓

로그 분석

↓

이상 탐지

↓

위협 대응

# 4. UML (Unified Modeling Language)

객체지향 시스템을 모델링하기 위한 **표준 모델링 언어**이다.

## UML 구성 요소

1. 사물(Thing)
2. 관계(Relationship)
3. 다이어그램(Diagram)

### 시험 암기

```
사물

↓

관계

↓

다이어그램
```

---

# 5. UML 관계(Relationship)

| 관계 | 의미 |
|------|------|
| 연관(Association) | 객체 간 연결 |
| 의존(Dependency) | 일시적으로 사용 |
| 일반화(Generalization) | 상속 |
| 실체화(Realization) | 인터페이스 구현 |

---

# 6. UML 클래스 다이어그램(Class Diagram)

객체의 구조를 표현하는 다이어그램이다.

표현

- 클래스
- 속성
- 메서드
- 클래스 간 관계

### 특징

- 가장 많이 출제
- 정적인 구조 표현

---

# 7. UML 인터페이스(Interface)

클래스가 반드시 구현해야 하는 **오퍼레이션(메서드)의 집합**을 정의한다.

### 특징

- 구현 내용 없음
- 기능만 정의
- 다형성 지원

### 시험 암기

```
인터페이스

=

구현해야 할 기능의 집합
```

---

# 8. ERD (Entity Relationship Diagram)

데이터베이스를 설계하기 위한 모델이다.

구성 요소

- 개체(Entity)
- 관계(Relationship)
- 속성(Attribute)

---

# 9. ERD 구성 요소

## 개체(Entity)

실제 관리 대상

예)

```
학생

직원

상품
```

---

## 관계(Relationship)

개체와 개체의 연결

예)

```
학생

↓

수강

↓

과목
```

---

## 속성(Attribute)

개체가 가지는 정보

예)

```
학번

이름

전화번호
```

---

# 10. ERD 시험 빈출 용어

## 실선

개체집합 ↔ 관계집합 연결

예)

```
학생 ─── 수강
```

---

## 관계집합

개체집합과 개체집합을 연결하는 관계

예)

```
학생

↓

수강

↓

과목
```

---

## 점선

관계집합 ↔ 관계집합의 속성 연결

---

## 관계집합의 속성

관계 자체가 가지는 속성

예)

```
학생

↓

수강

↓

신청일
```

신청일은 학생의 속성이 아니라

수강이라는 관계의 속성이다.

---

## 개체집합

같은 속성을 공유하는 개체들의 집합

예)

```
학생

직원

상품
```
