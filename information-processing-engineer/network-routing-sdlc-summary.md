✅ 1. 라우팅 프로토콜 (Routing Protocol)

👉 정의

네트워크에서 목적지까지 최적 경로를 찾는 규칙
라우터가 서로 정보 교환해서 가장 효율적인 경로 선택
🔹 (1) RIP

👉 거리 벡터 라우팅 프로토콜

✔ 특징

거리(홉 수) 기준으로 경로 선택
인접 라우터와 주기적 라우팅 테이블 교환
알고리즘: 벨만-포드 (Bellman-Ford)
최대 홉 수: 15 (16이면 도달 불가)

✔ 단점

느림 (수렴 속도 느림)
루프 발생 가능

✔ 한 줄 암기
👉 “RIP = 거리(홉) + 주기적 교환 + 최대 15”

🔹 (2) OSPF

👉 링크 상태 라우팅 프로토콜

✔ 특징

네트워크 전체 구조를 파악해서 경로 계산
알고리즘: 다익스트라 (Dijkstra)
메트릭: 비용(cost) (대역폭 기반)

✔ 장점

빠른 수렴
대규모 네트워크에 적합

✔ 한 줄 암기
👉 “OSPF = 링크 상태 + 다익스트라 + 비용 기반”

🔥 RIP vs OSPF 핵심 비교
구분	RIP	OSPF
방식	거리 벡터	링크 상태
기준	홉 수	비용
알고리즘	벨만-포드	다익스트라
속도	느림	빠름
규모	소규모	대규모
✅ 2. WAN (원거리 통신망)

👉 멀리 떨어진 지역 간 통신

🔹 (1) 전용회선 방식 (Leased Line)

✔ 특징

1:1 전용 연결 (점대점)
항상 연결된 상태
안정적, 보안성 높음

✔ 사용 프로토콜

PPP
HDLC

✔ 예시
👉 은행 본점 ↔ 지점 전용선

🔹 (2) 회선 교환 방식 (Circuit Switching)

✔ 특징

통신 전에 경로를 미리 설정
통신 동안 동일 경로 사용
대역폭 고정

✔ 대표 예시
👉 전화 통화

✔ 관련 기술

ISDN
🔹 (3) 패킷 교환 방식 (Packet Switching) ⭐ 중요

✔ 특징

데이터를 패킷 단위로 분할
각 패킷이 독립적으로 전송
서로 다른 경로로 갈 수 있음

✔ 장점

효율적 (회선 공유)
장애 발생 시 우회 가능

✔ 단점

지연 발생 가능
순서 바뀔 수 있음

✔ 예시
👉 인터넷 (웹, 카카오톡, 유튜브)

✔ 쉽게 이해
👉

회선교환: “한 번 길 잡으면 끝까지 같은 길”
패킷교환: “택배 여러 개로 나눠서 각자 다른 길로 감”
✅ 3. WSDL
🔹 WSDL

✔ 정의

웹서비스에 대한 설명서
XML 기반

✔ 포함 정보

서비스 위치
메시지 형식
프로토콜 정보

✔ 구성

SOAP + XML

✔ 한 줄 암기
👉 “WSDL = 웹서비스 설명서 (XML)”

✅ 4. REVOKE

✔ 정의

사용자 권한 제거

✔ 문법

REVOKE 권한 ON 테이블 FROM 사용자;

✔ 예시

REVOKE SELECT ON EMP FROM USER1;
✅ 5. 소프트웨어 생명주기 (SDLC)

✔ 순차적 진행 (뒤로 못 돌아감)

순서:
👉 타당성 → 계획 → 요구사항 → 설계 → 구현 → 테스트 → 유지보수

✔ 특징

문서 중심
변경 어려움
🔹 (2) 프로토타이핑 모델

✔ 특징

미리 시제품 만들어서 요구사항 확인

✔ 장점

사용자 요구 파악 쉬움
🔹 (3) 나선형 모델

✔ 특징

위험 분석 중심 반복 개발

순서:
👉 계획 → 위험분석 → 개발 → 고객평가

✔ 한 줄 암기
👉 “나선형 = 위험 관리”

🔹 (4) 반복적 모델 (Iterative)

✔ 특징

기능을 나눠서 반복적으로 개발

✔ 장점

점진적 개선
🔥 최종 핵심 암기 포인트
RIP 👉 거리벡터 / 홉수 / 15 제한
OSPF 👉 링크상태 / 다익스트라 / 비용
패킷교환 👉 나눠서 보냄 / 경로 다름
회선교환 👉 경로 고정
전용회선 👉 1:1 연결
WSDL 👉 웹서비스 설명 XML
REVOKE 👉 권한 제거
