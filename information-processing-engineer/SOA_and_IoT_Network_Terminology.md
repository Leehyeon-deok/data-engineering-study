1️⃣ SOA (Service-Oriented Architecture, 서비스 지향 아키텍처)
🔹 정의

SOA는 비즈니스 기능을 서비스 단위로 분리하여
**서로 느슨하게 결합(loose coupling)**된 구조로 시스템을 설계하는 아키텍처이다.

각 서비스는 독립적으로 배포·재사용 가능하며,
표준 인터페이스를 통해 통신한다.

🔹 SOA의 특징

재사용성: 동일 서비스 여러 시스템에서 사용 가능

느슨한 결합: 서비스 간 의존성 최소화

표준 기반: XML, SOAP, REST, WSDL 등 사용

플랫폼 독립성: OS, 언어에 종속되지 않음

🔹 SOA 구성 요소

Service Provider: 서비스 제공자

Service Consumer: 서비스 요청자

Service Registry: 서비스 목록 저장 및 검색 (UDDI)

🔹 장점 / 단점

장점

시스템 확장 및 변경 용이

유지보수성 향상

대규모 분산 시스템에 적합

단점

설계 복잡도 증가

성능 오버헤드 발생 가능

🔹 정처기 실기 포인트 ⭐

“느슨한 결합, 재사용성, 서비스 단위” 키워드 나오면 SOA

2️⃣ 데크 (DECT: Digital Enhanced Cordless Telecommunications)
🔹 정의

DECT는 무선 디지털 통신 표준으로
주로 무선 전화기, 사내 무선 통신 시스템에 사용된다.

🔹 특징

단거리 무선 통신

음성 통화 품질 우수

주파수 대역: 1.9GHz

다수의 단말 동시 연결 가능

🔹 활용 분야

무선 전화기

사무실·병원·공장 내부 통신

🔹 정처기 포인트 ⭐

“무선 전화, 사내 통신, DECT” → 데크

3️⃣ 피코넷 (Piconet)
🔹 정의

피코넷은 블루투스(Bluetooth) 기반의 소규모 무선 네트워크이다.

🔹 구조

Master 1대

Slave 최대 7대

Master 중심 통신 구조

🔹 특징

근거리 무선 통신

저전력

일시적 네트워크 구성 가능

🔹 예시

스마트폰 + 블루투스 이어폰

노트북 + 마우스 + 키보드

🔹 정처기 포인트 ⭐

“블루투스, Master/Slave, 소규모 네트워크”

4️⃣ 지그비 (ZigBee)
🔹 정의

지그비는 저전력·저속 데이터 전송을 위한
근거리 무선 통신 기술로, IoT 환경에 최적화되어 있다.

🔹 특징

저전력 소비 (배터리 수년 사용 가능)

전송 속도 느림

Mesh 네트워크 지원

IEEE 802.15.4 기반

🔹 활용 분야

스마트홈

센서 네트워크

산업 자동화

IoT 디바이스

🔹 비교 (피코넷 vs 지그비)
구분	피코넷	지그비
기반	블루투스	IEEE 802.15.4
전력	중간	매우 낮음
구조	Master-Slave	Mesh
용도	개인 기기	IoT 센서
🔹 정처기 포인트 ⭐

“저전력, IoT, 센서 네트워크” → 지그비

5️⃣ 정형 명세 기법 (Formal Specification)
🔹 정의

정형 명세 기법은
수학적·논리적 표현을 사용하여 시스템을 명확하고 엄밀하게 명세하는 기법이다.

🔹 목적

요구사항의 모호성 제거

설계 오류 사전 검출

시스템의 정확성·일관성 보장

🔹 특징

수학적 기호 사용

형식적 검증 가능

자동 검증 도구 활용 가능

🔹 대표 기법

Z 기법

VDM

Petri Net

B-Method

🔹 장단점

장점

오류 최소화

고신뢰 시스템에 적합

단점

학습 난이도 높음

비용 및 시간 증가

🔹 정처기 포인트 ⭐

“수학적 명세, 형식적 검증, 요구사항 명확화”

🎯 시험 직전 요약 한 줄

SOA: 서비스 단위, 느슨한 결합, 재사용성

DECT(데크): 무선 전화 통신

피코넷: 블루투스 기반 소규모 네트워크

지그비: 저전력 IoT 통신

정형명세기법: 수학적·형식적 시스템 명세
