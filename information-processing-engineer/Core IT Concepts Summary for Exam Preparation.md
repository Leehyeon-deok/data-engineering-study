✅ 1. 진수 표현 (암기 포인트)
🔹 C / Java
0 → 8진수
0x → 16진수
0b → 2진수 (Java만 가능, C는 X)
🔹 Python
0b → 2진수
0o → 8진수 (C/Java랑 다름 주의)
0x → 16진수

👉 핵심 암기

Python만 8진수 = 0o
Java만 0b 가능 (C는 안됨)
✅ 2. 비트 연산 (시험 핵심)
연산자	의미	핵심
&	AND	둘 다 1 → 1
|	OR	하나라도 1 → 1
^	XOR	다르면 1
~	NOT	반전
<<	왼쪽 시프트	×2 효과
>>	오른쪽 시프트	÷2 효과

👉 한 줄 암기

XOR = 다르면 1
SHIFT = 비트 이동 = 곱/나눗셈 효과
✅ 3. 템퍼프루핑 (Tamper Proofing)

👉 핵심 정의

외부 공격으로부터 소프트웨어 위·변조 방지 + 감지 기술

👉 특징

변조 감지 시 프로그램 일부러 오작동
🔹 구성 기술
해시 함수 → 무결성 검증
워터마크 → 저작권 삽입
핑거프린트 → 사용자 추적
코드 난독화 → 분석 어렵게

👉 한 줄 암기
👉 “위변조 막고, 걸리면 망가뜨림”

✅ 4. 디자인 패턴
🔹 MVC 패턴

👉 구조

Model (데이터)
View (화면)
Controller (제어)

👉 핵심

UI와 로직 분리 → 유지보수 쉬움
🔹 파이프-필터 패턴

👉 흐름

입력 → 처리 → 다음으로 전달 반복

👉 핵심

데이터가 필터를 통과하면서 계속 가공됨

👉 예시

스트림 처리, 로그 처리

✅ 6. 법칙 정리 (시험 단골🔥)
🔹 브룩스의 법칙

👉

늦어진 프로젝트에 인력 추가 → 더 늦어짐

👉 이유

의사소통 비용 증가
🔹 파레토 법칙 (80:20 법칙)

👉

결과 80% ← 원인 20%

👉 예시

매출 80% ← 고객 20%
🔹 롱테일 법칙 (파레토 반대)

👉

소수 핵심보다 다수의 사소한 것들이 더 큰 가치

👉 예시

넷플릭스: 인기작보다 비인기 콘텐츠 전체
✅ 💥 최종 핵심 암기 세트

👉 이렇게 외우면 시험 바로 맞춘다

파레토 → 20%가 80% 만든다
롱테일 → 80%가 더 중요하다
브룩스 → 사람 늘리면 더 늦어진다
MQTT → 브로커 + 발행/구독
CoAP → REST + IoT
템퍼프루핑 → 변조 막고 걸리면 망가뜨림
