# 데이터 수집 개요

## 1. 데이터 수집 수행 절차
데이터 수집은 비즈니스 이해를 기반으로 단계적으로 수행된다.

1. 비즈니스 도메인 정보 수집  
2. 분석 기획서 기반 도메인 및 서비스 이해  
3. 수집 대상 데이터 탐색  
4. 기초 데이터 수집

→ 수집 단계에서 도메인 이해가 부족하면 분석 품질이 저하될 수 있다.

---

## 2. 데이터 수집 시스템 구축 절차

1. 수집 데이터 유형 파악  
2. 수집 기술 결정  
3. 수집 아키텍처 수립  
4. 하드웨어 구축  
5. 실행 환경 구축

→ 데이터 특성에 따라 배치 / 스트리밍 수집 방식이 달라진다.

---

## 3. 데이터 유형별 수집 방식 및 기술

### 3.1 정형 데이터
- ETL
- FTP
- API
- DB to DB
- Sqoop

→ 관계형 데이터베이스 기반 데이터

### 3.2 비정형 데이터
- 크롤링
- RSS
- Open API
- Kafka
- Flume

→ 로그, 텍스트, 이미지 등 구조가 없는 데이터

### 3.3 반정형 데이터
- Flume
- Scribe
- 센싱 데이터
- 스트리밍 데이터

→ JSON, XML 형태의 유연한 구조 데이터

## 4. 개인정보 비식별화 및 적정성 평가

### 4.1 k-익명성 (k-Anonymity)
- 특정 개인이 k명 중 하나로 식별되도록 보장
- 문제점: 민감 정보가 동일하면 추론 가능

### 4.2 l-다양성 (l-Diversity)
- 민감 속성이 여러 종류로 존재하도록 보완
- 문제점:
  - 의미적 유사성 문제
  - 분포 왜곡 문제

### 4.3 t-근접성 (t-Closeness)
- 민감 정보의 분포가 전체 데이터 분포와 유사하도록 유지
- 단점:
  - 구현 복잡
  - 데이터 유용성 감소 가능
