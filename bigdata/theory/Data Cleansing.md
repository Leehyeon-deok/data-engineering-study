# 데이터 정제 (Data Cleansing)

## 1. 데이터 이해

### 1) 데이터 관련 정의
- 데이터 단위
- 관측값
- 변수
- 원자료(Raw Data)

### 2) 데이터의 종류

#### (1) 변수 개수 기준
- 단변량 자료  
  → 특성 변수가 1개인 자료
- 다변량 자료  
  → 특성 변수가 2개 이상인 자료

#### (2) 자료 성격 기준
- 질적 자료 (정성적 / 범주형)
  - 명목자료: 범주·종류 구분 (숫자는 의미 없음)
  - 서열자료: 순서 의미는 있으나 간격 의미 없음
- 수치 자료 (정량적 / 연속형)
  - 구간자료: 산술 연산 가능 (비율 개념 없음)
  - 비율자료: 절대적 0 존재, 비율 계산 가능

#### (3) 시간 기준
- 시계열 자료
- 횡단면 자료(횡적자료): 단일 시점, 여러 대상
- 종적자료(패널데이터): 여러 대상 + 여러 시점

---

## 2. 데이터 정제

### 1) 정의
- 수집된 데이터 중 분석에 필요한 데이터를 추출·통합하는 과정

### 2) 필요성
- 분석 도구와 기법에 맞게 데이터 가공
- 분석 결과의 신뢰성 확보

### 3) 정제하지 않은 데이터의 문제점
- 데이터 일관성 부족
- 분석 처리 어려움
- 결과 신뢰도 저하

### 4) 정제 과정
- 데이터 품질 확인
- 비정형 → 정형 데이터 변환
- 결측치 및 오류 수정
- 레거시 시스템 데이터와 통합 가능

---

## 3. 데이터 집계 · 변환

### 1) 데이터 집계
- 합계, 평균, 중앙값, 최빈값 등으로 요약
- 데이터 특성 파악 목적

### 2) 데이터 변환
- 일반화: 패턴 파악, 복잡성 감소
- 정규화: 범위 조정, 상대적 크기 제거, 이상치 영향 완화
- 평활화: 노이즈 제거, 추세·패턴 부드럽게 표현

---

## 4. 데이터 정제의 전·후처리

- 전처리: 저장 전 처리
  - 데이터 대상·수집 방법 결정
  - 저장 방식 및 장소 선정
- 후처리: 저장 후 처리
  - 데이터 품질 관리

---

## 5. 결측값 처리

### 1) 결측치 정의
- 데이터 값이 존재하지 않는 상태

### 2) 잘못된 처리 시 문제
- 임의 제거: 분석 유의성 감소
- 임의 대체: 편향 발생, 신뢰성 저하

### 3) 결측 데이터 유형
- MCAR (완전 무작위 결측)
- MAR (무작위 결측)
- MNAR (비무작위 결측)

### 4) 결측 데이터 분석 시 고려사항
- 효율성
- 처리 복잡성
- 편향 발생 가능성

---

## 6. 결측치 대치 방법

### 1) 단순 대치법
- 완전분석: 결측 데이터 제거
- 평균 대치법
  - 장점: 효율성 증가
  - 단점: 표준오차 과소추정
- 회귀 대치법
  - 조건부 평균 대치
  - 다른 변수로 회귀모델 구축
- 단순확률 대치법
  - 평균 대치법 단점 보완
  - 무작위 추출로 대치
- 최근접 대치법
  - 유사 응답값으로 대치
  - 동일 값 반복 사용 가능성

### 2) 다중 대치법
- 단순 대치를 여러 번 수행
- 단계: 대치 → 분석 → 결합
- 통계적 효율성과 일치성 우수

---

## 7. 이상값(Outlier) 처리

### 1) 이상치 정의
- 정상 범주를 벗어난 값

### 2) 이상치 종류
- 단변수 이상치
- 다변수 이상치

### 3) 발생 원인
- 비자연적: 입력 오류, 측정 오류, 실험 오류, 처리 오류
- 자연적: 실제 현상에 의한 극단값

### 4) 문제점
- 평균·분산 왜곡
- 회귀·검정 등 고급 분석 신뢰성 저하
- 데이터 신뢰도 감소

---

## 8. 이상치 탐지 방법

### 1) 시각화 방법 (비모수적)
- 상자수염그림(Box Plot)
- 줄기-잎 그림
- 산점도

### 2) Z-score (모수적 단변량)
- 평균과의 거리(표준편차 단위)
- 정규화 후 계산

### 3) 밀도 기반 클러스터링
- 밀도 낮은 영역을 이상치로 판단
- 비모수적 다변량 데이터에 적합

### 4) 고립 의사나무(Isolation Tree)
- 데이터 고립 정도로 이상치 판단
