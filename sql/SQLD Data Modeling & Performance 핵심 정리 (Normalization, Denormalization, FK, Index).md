✅ 1. 정규화 (Normalization)
✔ 개념

정규화는 데이터 중복을 제거하고 데이터 무결성을 유지하기 위해 수행한다.

👉 데이터 모델이 무결성을 보장하기 어려운 경우 정규화를 수행한다.

✔ 목적

데이터 중복 제거

데이터 무결성 유지

이상(Anomaly) 방지

삽입 이상

삭제 이상

갱신 이상

✔ 정규화 단계 핵심
단계	설명
1NF	원자값
2NF	부분 함수 종속 제거
3NF	이행 종속 제거
BCNF	결정자 후보키
✔ 시험 핵심 포인트

👉 정규화 단계에서
주식별자와 관련성이 가장 낮은 속성은 분리 대상이다.

✔ 예시
❌ 정규화 전
주문번호	고객명	고객주소	상품
1	홍길동	서울	노트북

👉 고객주소는 주문과 직접 관련성이 낮음

✅ 정규화 후

고객 테이블
| 고객ID | 고객명 | 고객주소 |

주문 테이블
| 주문번호 | 고객ID | 상품 |

✅ 2. 반정규화 (Denormalization)
✔ 개념

조회 성능 향상을 위해 의도적으로 중복을 허용하는 것.

✔ 반정규화가 필요한 이유

JOIN 감소 → 조회 속도 향상

집계 성능 향상

OLAP 환경에서 중요

데이터 조회가 많은 경우

👉 즉, 성능 개선이 목적

✔ 반정규화 시 주의

데이터 무결성 위험 증가

데이터 중복 관리 필요

✔ 예시

고객과 주문을 JOIN하지 않도록 고객명을 주문 테이블에 추가.

✅ 3. 반정규화 방법
✔ ① 수직 분할

👉 컬럼 단위 분리

자주 조회하는 컬럼과 그렇지 않은 컬럼 분리

I/O 감소

예)

회원 기본정보

회원 상세정보

✔ ② 수평 분할

👉 행 단위 분리

데이터 양이 많은 경우

기간별, 지역별 분리

예)

2024 주문

2025 주문

✔ ③ 중복 추가

👉 조회 성능 향상 목적

👉 관계의 반정규화
데이터 무결성을 깨뜨릴 위험 없이 조회 성능 향상 가능.

✅ 4. 기능성 컬럼 추가

👉 최신 데이터 조회 성능 개선.

예)

변경일 기준 최신 데이터 조회 → 정렬 필요

최신 여부 컬럼 추가 → 정렬 없이 조회 가능.

✔ 예시
ID	변경일	최신여부
1	2024	N
1	2025	Y

👉 WHERE 최신여부 = 'Y'

✅ 5. FK (Foreign Key)
✔ 개념

다른 테이블의 PK를 참조하는 컬럼.

👉 관계 무결성 유지.

✔ 위치

👉 FK는 항상 M(다)쪽에 존재한다.

✔ 인덱스 생성 이유

JOIN 성능 향상

DELETE, UPDATE 성능 향상

Lock 감소

👉 FK 컬럼에 인덱스 생성은 매우 중요.

✔ 예시

고객(1) : 주문(M)

주문 테이블에 고객ID(FK) 존재.
