1️⃣ 성능 데이터 모델링의 정의

성능 데이터 모델링이란
👉 데이터 모델링 단계에서부터 성능을 고려하여
👉 조회·입력·수정·삭제 성능을 향상시키기 위한 모델링 기법이다.

정규화 / 반정규화

테이블 통합 / 분할

컬럼 추가, 중복 허용
등을 통해 업무 성능 최적화가 목적

2️⃣ 이상(Anomaly) 현상

정규화가 안 되었을 때 발생하는 문제

🔹 삭제 이상 (Deletion Anomaly)

어떤 정보를 삭제했을 때

의도하지 않은 다른 정보까지 함께 삭제됨

예) 수강 취소 → 학생 정보까지 삭제

🔹 삽입 이상 (Insertion Anomaly)

원하는 정보를 입력하려 해도

관련 없는 다른 정보가 없으면 입력 불가

예) 학생 정보 입력하려면 수강 정보도 필요

🔹 갱신 이상 (Update Anomaly)

동일한 데이터가 여러 곳에 중복 저장되어

일부만 수정되어 데이터 불일치 발생

3️⃣ 정규화 (Normalization)
✔ 정의

주제별로 독립된 테이블로 분리

중복 제거

데이터 무결성 & 일관성 유지

✔ 목적

최소한의 데이터 중복

최대한의 데이터 유연성

데이터 일관성 유지

모델의 독립성 확보

👉 한 문장 요약
“데이터 중복을 제거하고 테이블 간 독립성을 확보하는 과정”

4️⃣ 함수 종속 (시험 핵심)
🔹 도부이결다조 (정처기 암기어)

도메인
부분함수종속
이행함수종속
결정자
다중값 종속
조인 종속

(시험에서 단답으로 자주 등장)

🔹 부분 함수 종속 (Partial Functional Dependency)

복합 PK 중 일부만으로 속성이 결정

2NF 위반

📌 예
PK = (학번, 과목번호)
학번 → 학생이름 ❌ (부분 함수 종속)

👉 해결: 테이블 분리

🔹 이행 함수 종속 (Transitive Dependency)

PK가 아닌 속성이 다른 속성을 결정

3NF 위반

📌 예
학번 → 학과코드 → 학과명
(학번 → 학과명은 이행 종속)

👉 해결: 학과 테이블 분리

5️⃣ 정규화와 성능
✔ 장점

데이터 중복 감소

저장 공간(용량) 최소화

데이터 무결성 유지

입력/수정/삭제 안정성 향상

❌ 단점

테이블이 많아짐

JOIN 증가

조회 성능 저하 가능

👉 시험 포인트
“정규화는 무결성 ↑, 조회 성능은 ↓ 가능”

6️⃣ 반정규화 (Denormalization)
✔ 정의

DB 성능 향상을 위해

의도적으로 데이터 중복을 허용

JOIN을 줄이는 기법

✔ 특징

조회 성능 ↑

JOIN 감소

테이블 구조 단순화

❌ 단점

데이터 중복 증가

데이터 불일치 가능성

입력 / 수정 / 삭제 성능 저하

무결성 관리 어려움

👉 한 줄 요약
“조회 성능을 얻고, 무결성을 희생”

7️⃣ 정규화 vs 반정규화 한눈 비교
구분	정규화	반정규화
목적	무결성, 일관성	조회 성능
데이터 중복	최소	허용
JOIN	많음	적음
조회 성능	상대적 저하	향상
입력/수정/삭제	안정적	성능 저하
