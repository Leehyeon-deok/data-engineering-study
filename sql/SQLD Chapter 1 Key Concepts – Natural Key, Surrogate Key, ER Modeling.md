✅ 1. 식별자 (Identifier)

식별자는 엔터티 내에서 각각의 인스턴스를 유일하게 구분하는 속성이다.
즉, 하나의 데이터를 다른 데이터와 구별하는 기준이다.

✔ 식별자의 분류 (대체 여부 기준)
🔷 ① 본질식별자 (Natural Key)

👉 업무에 원래 존재하는 식별자
👉 비즈니스 프로세스에 의해 생성되는 식별자

예시

주민등록번호

학번

사업자등록번호

즉, 업무 자체에서 의미를 가지는 식별자

✔ 특징

의미 있음

변경 가능성 존재

현실 세계에서 유래

🔷 ② 인조식별자 (Surrogate Key)

👉 시스템에서 인위적으로 만든 식별자
👉 대체식별자라고도 한다.

예시

고객ID

주문번호

자동 증가 값 (Sequence)

CUSTOMER_ID NUMBER PRIMARY KEY;
✔ 특징

의미 없음

변경 거의 없음

성능, 관리에 유리

📌 시험 핵심

👉 본질식별자 = 업무에서 생성
👉 인조식별자 = 시스템에서 생성

✅ 2. 관계(Relationship) 도출

관계는 엔터티 간의 연관성을 의미한다.

✔ 관계 도출 방법
🔷 ① 업무 규칙에서 동사 표현

👉 관계는 동사에서 도출

예시

고객이 주문한다

학생이 수강한다

직원이 부서를 관리한다

👉 “한다”, “관리한다” → 관계

🔷 ② 정보의 조합 발생

👉 공통 기준키를 통해 조인이 필요한 경우
👉 관계가 존재한다고 판단

예시

주문 + 고객 정보 조회

직원 + 부서 정보 조회

👉 공통 키 필요 → 관계 존재

📌 핵심 정리

✔ 관계 → 동사
✔ 엔터티 → 명사

🖼️ 관계 도출 예시
4
✅ 3. 관계의 3가지 요소

관계를 표현할 때 반드시 고려해야 하는 요소

✔ ① 관계명

관계의 의미

동사 형태

예시

주문한다

소속된다

관리한다

✔ ② 관계 차수 (Cardinality)

엔터티 간의 관계 개수

예시

1 : 1

1 : N

N : M

✔ ③ 관계 선택사양 (Optionality)

참여 여부

예시

필수 관계

선택 관계

📌 시험 포인트
👉 관계 = 관계명 + 관계차수 + 선택사양

🖼️ 관계 요소 이해

✅ 4. 주식별자 (Primary Identifier)

엔터티를 유일하게 식별하는 대표 식별자이다.

✔ 특징

✔ 유일성 (Unique)
👉 모든 인스턴스를 구분

✔ 최소성 (Minimality)
👉 최소 개수로 구성

✔ 불변성
👉 변경 최소화

✔ 존재성
👉 NULL 불가

✔ 최소성 의미

👉 불필요한 속성을 제거
👉 최소 개수로 식별

예시

학번만으로 식별 가능하면

이름 + 학번 ❌

📌 시험 핵심
👉 유일성 + 최소성 = 주식별자

🖼️ Primary Key 이해
4
✅ 5. 속성(Attribute)

속성은 엔터티가 가지는 데이터 항목이다.

🔷 파생속성 (Derived Attribute)

👉 다른 속성으로부터 계산되는 속성
👉 규칙이나 공식에 의해 생성

예시

나이 = 현재날짜 – 생년월일

총금액 = 수량 × 단가

✔ 특징

저장하지 않는 경우 많음

성능 문제로 저장하기도 함

📌 시험 포인트
👉 계산 결과 → 파생속성

🖼️ 파생속성 이해
