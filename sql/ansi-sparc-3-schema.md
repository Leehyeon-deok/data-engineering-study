# ANSI-SPARC 3단계 스키마 구조

## 개요
ANSI-SPARC 아키텍처는 데이터베이스 구조를 3단계로 분리하여
데이터 독립성을 보장하기 위한 표준 구조이다.

---

## 1. 외부 스키마 (External Schema)
- 사용자 관점의 데이터베이스 구조
- 사용자 또는 응용 프로그램이 보는 뷰(View)
- 사용자마다 서로 다른 외부 스키마를 가질 수 있음

### 담당 역할
- 사용자 (User)

---

## 2. 개념 스키마 (Conceptual Schema)
- 전체 데이터베이스의 논리적 구조
- 모든 사용자에게 공통
- 데이터 간의 관계, 제약 조건을 정의

### 담당 역할
- 데이터베이스 설계자 (DBA, 설계자)

---

## 3. 내부 스키마 (Internal Schema)
- 물리적 저장 구조
- 파일 구성, 인덱스, 접근 경로 정의
- 실제 디스크에 어떻게 저장되는지 표현

### 담당 역할
- 개발자 / 시스템 설계자

---

## 데이터 독립성

### 논리적 데이터 독립성
- 개념 스키마 변경 → 외부 스키마 영향 없음

### 물리적 데이터 독립성
- 내부 스키마 변경 → 개념 스키마 영향 없음

---

## 시험 & 면접 포인트
- ANSI-SPARC 구조의 목적은 **데이터 독립성 확보**
- 외부/개념/내부 스키마의 역할 구분 중요
- SQLD 및 정보처리기사 단골 출제 주제
