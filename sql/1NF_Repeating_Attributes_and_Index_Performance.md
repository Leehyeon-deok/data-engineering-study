📘 1차 정규화와 반복 속성, OR 조건, 인덱스 성능 이슈
1️⃣ 반복 속성과 제1정규형(1NF)
제1정규형(1NF) 정의

모든 속성은 원자값(Atomic Value) 을 가져야 한다.

하나의 컬럼에 반복적 속성값이 존재하면 1NF 위반이다.

반복 속성 예시
과목1, 과목2, 과목3


또는

전화번호1 OR 전화번호2 OR 전화번호3


➡️ 동일한 의미의 속성이 컬럼 단위로 반복됨

2️⃣ 반복 속성과 OR 조건 문제
반복 속성 사용 시 검색 방식

반복 속성이 컬럼으로 존재하면 조건 검색 시 다음과 같은 형태가 된다.

WHERE col1 = 'A'
   OR col2 = 'A'
   OR col3 = 'A'

OR 조건의 성능 문제

OR로 연결된 조건 중 하나라도 인덱스가 없으면

👉 모든 조건이 인덱스를 사용하지 못함

👉 전체 테이블 스캔(Full Table Scan) 발생

성능 저하 원인

옵티마이저가 인덱스 선택을 포기

데이터 양이 많을수록 성능 급격히 저하

📌 시험 포인트

OR 조건 사용 시 하나의 컬럼이라도 인덱스가 없으면
전체 데이터 스캔으로 성능 저하 발생

3️⃣ 반복 속성에 인덱스를 생성하면?
장점 (조회 성능)

조건 검색 시 인덱스 활용 가능

SELECT 성능 향상

단점 (쓰기 성능)

INSERT / UPDATE / DELETE 시

인덱스도 함께 수정 필요

쓰기 성능 저하 발생

📌 인덱스가 많을수록

검색 성능 ⬆

입력/수정/삭제 성능 ⬇

4️⃣ 정규화를 통한 해결 (1NF 적용)
반복 속성 제거

반복 컬럼 → 행(Row) 단위로 분리

❌ 비정규화 구조

학생(학번, 과목1, 과목2, 과목3)


⭕ 1NF 적용

수강(학번, 과목)

정규화 효과

OR 조건 제거

단일 컬럼 조건 검색 가능

인덱스 효율 극대화

WHERE 과목 = 'DB'


➡️ 인덱스 정상 사용
➡️ 성능 안정적

5️⃣ 정규화 vs 반복 속성 + 인덱스 비교
구분	반복 속성	1NF 적용
구조	컬럼 반복	행 분리
검색 조건	OR 조건 다수	단일 조건
인덱스 활용	제한적	효율적
조회 성능	불안정	안정적
쓰기 성능	인덱스 많아 저하	균형
6️⃣ 시험용 핵심 문장 정리

반복 속성은 제1정규형 위반

OR 조건 사용 시 하나라도 인덱스가 없으면 전체 스캔

반복 속성에 인덱스 생성 시

조회 성능은 향상

입력/수정/삭제 성능은 저하

정규화를 통해 OR 조건 제거 및 성능 개선 가능

7️⃣ 한 줄 요약 (정처기 실기 스타일)

반복 속성은 1차 정규화 대상이며,
OR 조건 사용 시 인덱스 활용이 제한되어 성능 저하가 발생한다.
반복 속성에 인덱스를 생성하면 조회 성능은 향상되나
입력·수정·삭제 성능은 저하된다.
