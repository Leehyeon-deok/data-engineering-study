📘 데이터베이스(Database) 기본 개념 정리
1. 데이터베이스(DB)란?

DB란 일정한 규칙(스키마)에 따라 구조화되어 저장되는 데이터의 집합

데이터를 통합 관리하고 중복을 최소화

실시간 접근과 동시 공유 가능

2. DBMS (Database Management System)

데이터베이스를 정의·조작·제어하는 소프트웨어

DBMS가 제공하는 질의 언어(SQL) 를 통해

삽입(INSERT)

삭제(DELETE)

수정(UPDATE)

조회(SELECT)

DB 위에 존재하며 응용 프로그램과 DB 사이의 중간 계층

계층 구조
응용 프로그램
     ↓
   DBMS
     ↓
   Database

3. 엔터티(Entity)
3.1 엔터티 정의

여러 개의 속성을 지닌 명사형 개체

업무 요구사항에 따라 속성이 결정

의미 없는 속성은 제거됨

3.2 강한 엔터티 / 약한 엔터티

강한 엔터티

독립적으로 존재 가능

자체 식별자 보유

약한 엔터티

단독 존재 불가

강한 엔터티에 종속

📌 예시

건물(강한 엔터티)

방(약한 엔터티)

4. 릴레이션(Relation)
4.1 릴레이션 정의

DB에서 데이터를 저장하는 논리적 기본 단위

엔터티의 데이터를 저장

관계형 DB에서는 테이블(Table)

4.2 관계형 DB vs NoSQL

관계형 DB

MySQL, Oracle

테이블 기반 (레코드, 컬럼)

NoSQL

MongoDB

컬렉션(Collection), 도큐먼트(Document)

📌 구조

레코드(튜플) → 테이블 → 데이터베이스

5. 속성(Attribute)

릴레이션에서 관리하는 구체적 정보

고유한 이름과 데이터 타입을 가짐

6. 도메인(Domain)

속성이 가질 수 있는 값의 범위

데이터 타입 + 길이 + 제약 조건

7. 필드(Field)와 레코드(Record)
7.1 레코드

테이블의 행(row)

하나의 인스턴스

튜플(Tuple)이라고도 함

7.2 필드

테이블의 열(column)

속성이 실제 저장되는 공간

7.3 데이터 타입

숫자형: INT, BIGINT

문자형:

CHAR: 고정 길이

VARCHAR: 가변 길이

날짜형: DATE, DATETIME

대용량 데이터:

TEXT: 큰 문자열

BLOB: 이진 데이터

열거형:

ENUM: 단일 선택

SET: 다중 선택

8. 테이블 간 관계(Relationship)

여러 테이블은 서로 관계를 가짐

관계 유형

1 : 1

1 : N

N : M
→ 관계형 DB에서는 중간 테이블로 해소

9. 키(Key)
9.1 키의 종류
① 슈퍼키(Super Key)

유일성을 만족하는 속성 집합

② 후보키(Candidate Key)

유일성 + 최소성 만족

③ 기본키(Primary Key)

후보키 중 대표로 선택된 키

④ 대체키(Alternate Key)

후보키 중 기본키로 선택되지 않은 키

⑤ 외래키(Foreign Key)

다른 테이블의 기본키 참조

9.2 기본키 유형
자연키(Natural Key)

업무적으로 자연스럽게 존재

시간이 지나면 변경 가능성 존재

인조키(Surrogate Key)

인위적으로 생성

값이 변하지 않음

실무에서 기본키로 주로 사용

🧠 시험 대비 핵심 요약

DB는 구조화된 데이터의 집합

DBMS는 DB를 관리하는 중간 계층

엔터티는 명사형 개체

릴레이션은 테이블

키는 유일성 / 최소성이 핵심

N:M 관계는 중간 테이블 필요
