# 네트워크 기초 정리

## 1. 네트워크(Network)란?

* **노드(Node)**와 **링크(Link)**가 서로 연결되어 **리소스를 공유**하는 집합
* **노드**: 서버, 라우터, 스위치 등 네트워크 장치
* **링크**: 노드를 연결하는 통신 매체 (유선 / 무선)

---

## 2. 좋은 네트워크의 조건

* **높은 처리량(Throughput)**
* **짧은 지연시간(Latency)**
* **낮은 장애 빈도**
* **우수한 보안성**

---

## 3. 네트워크 성능 지표

### 3.1 처리량 (Throughput)

* 링크 내에서 **성공적으로 전달된 데이터의 양**
* 보통 **얼마만큼의 트래픽을 처리했는지**를 의미
* 단위: **bps (bit per second)**

### 3.2 대역폭 (Bandwidth)

* **주어진 시간 동안** 네트워크 연결을 통해 **전송 가능한 최대 비트 수**
* 처리량의 *이론적 최대치*

### 3.3 지연시간 (Latency)

* 요청이 처리되는 데 걸리는 시간
* 메시지가 **두 장치 간 왕복(RTT)** 하는 데 걸린 시간
* 지연시간에 영향을 주는 요소

  * 매체 타입 (유선 / 무선)
  * 패킷 크기
  * 라우터의 패킷 처리 시간

---

## 4. 네트워크 토폴로지 (Topology)

* 노드와 링크가 **어떻게 배치·연결**되어 있는지를 나타내는 구조

### 4.1 트리(Tree) 토폴로지

* 계층형 구조
* **장점**

  * 노드 추가·삭제가 쉬움
* **단점**

  * 특정 상위 노드에 트래픽이 집중되면 하위 노드에 영향

### 4.2 버스(Bus) 토폴로지

* 하나의 중앙 통신 회선에 여러 노드가 연결된 구조
* 주로 **LAN**에서 사용
* **장점**

  * 설치 비용이 적음
  * 구조가 단순, 노드 추가·삭제 용이
* **단점**

  * **스푸핑(Spoofing)** 공격에 취약

> 스푸핑: LAN 환경에서 스위칭 기능을 속이거나 마비시켜, 패킷이 의도하지 않은 노드로 전달되게 하는 공격

### 4.3 스타(Star) 토폴로지

* 중앙 노드에 모든 노드가 연결된 구조
* **장점**

  * 노드 추가 및 장애 탐지 용이
  * 패킷 충돌 가능성 낮음
  * 중앙 노드가 아닌 노드의 장애는 전체에 영향 적음
* **단점**

  * 중앙 노드 장애 시 전체 네트워크 마비
  * 설치 비용이 상대적으로 높음

### 4.4 링(Ring) 토폴로지

* 각 노드가 양옆의 두 노드와 연결되어 고리 형태 구성
* **장점**

  * 노드 수가 증가해도 손실이 적음
  * 충돌 발생 가능성 낮음
  * 장애 노드 탐지 용이
* **단점**

  * 네트워크 구성 변경이 어려움
  * 회선 장애 시 전체 네트워크에 큰 영향

### 4.5 메시(Mesh) 토폴로지

* 노드 간 다중 경로로 연결된 구조 (망형)
* **장점**

  * 단일 노드 장애에도 네트워크 유지 가능
  * 트래픽 분산 처리 가능
* **단점**

  * 노드 추가가 어려움
  * 구축 및 운영 비용이 높음

---

## 5. 병목현상 (Bottleneck)

* 시스템 성능이나 용량이 **특정 구성 요소**로 인해 제한되는 현상

### 토폴로지가 중요한 이유

* 서비스에서 지연시간이 길게 발생
* 관리자가 대역폭을 증가시켰으나 성능 개선 없음
* 원인 분석 결과 **네트워크 토폴로지 구조상 병목 발생**
* 회선 추가, 구조 변경 등을 통해 병목 지점 해결

➡️ **결론**: 네트워크 성능 문제 해결 시 단순한 대역폭 증가뿐 아니라, **토폴로지 구조 분석이 필수**
