 DI (Dependency Injection, 의존성 주입)
1. 개념

의존성(Dependency)
→ A가 B에 의존한다 = B가 바뀌면 A도 함께 바뀐다.

DI란
→ 객체가 필요한 의존성을 직접 생성하지 않고,
→ **외부(상위 모듈)**에서 주입받는 방식

 직접 생성 (강한 결합)
A → new B()

 DI (느슨한 결합)
A → B를 주입받음

2. 목적

모듈 간 결합도 감소 (Decoupling)

테스트 용이성 증가

유지보수 및 교체 쉬움

3. 특징

상위 모듈과 하위 모듈이 **구현체가 아닌 추상화(인터페이스)**에 의존

객체 생성 책임을 외부로 분리

4. 장점

단위 테스트(TDD)에 유리

모듈 교체 및 마이그레이션 쉬움

코드 구조가 명확해짐

5. 단점

클래스 수 증가 → 구조 복잡

런타임 비용 증가 가능

설계 난이도 상승

6. DI 원칙 (DIP)

상위 모듈은 하위 모듈에 의존 ❌

둘 다 추상화에 의존해야 함

7. 언어별 특징

Java

타입 엄격, new 사용 → DI 어렵

Spring 같은 DI 컨테이너 사용

JavaScript

함수/객체 자유 → 자연스럽게 DI 가능
