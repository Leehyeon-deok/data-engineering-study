📚 Data Structure & Complexity 정리
1️⃣ 자료구조(Data Structure)
✅ 자료구조란?

데이터를 효율적으로 저장, 관리, 수정, 삭제, 탐색하기 위한 데이터의 집합 구조

자료구조는 단순히 데이터를 저장하는 것이 아니라
어떻게 저장해야 빠르게 접근하고 효율적으로 처리할 수 있는지를 고민하는 학문이다.

2️⃣ 복잡도 (Complexity)
📌 2-1. 시간 복잡도 (Time Complexity)
✔ 정의

알고리즘이 문제를 해결하는 데 걸리는 시간을 입력 크기 n에 대한 함수로 표현한 것

보통 빅오 표기법(Big-O Notation) 으로 나타낸다.

✔ 왜 필요한가?

알고리즘의 성능 비교

입력 크기가 커질 때 얼마나 느려지는지 예측

최적의 알고리즘 선택

📌 2-2. 빅오 표기법 (Big-O)

최악의 경우(Worst Case)를 기준으로 증가율을 표현

주요 시간복잡도
표기법	의미	예시
O(1)	상수 시간	배열 인덱스 접근
O(log n)	로그 시간	이진 탐색
O(n)	선형 시간	선형 탐색
O(n log n)	로그 선형	퀵/병합 정렬
O(n²)	이차 시간	이중 반복문
O(2ⁿ)	지수 시간	재귀 완전탐색
🔎 예시
# O(1)
arr[0]

# O(n)
for i in arr:
    print(i)

# O(n²)
for i in arr:
    for j in arr:
        print(i, j)

📌 2-3. 공간 복잡도 (Space Complexity)

프로그램 실행 시 필요한 메모리 공간의 양

포함 요소

정적 변수

동적 할당 메모리

재귀 호출로 인한 스택 메모리

예시
def recursive(n):
    if n == 0:
        return
    recursive(n-1)


👉 재귀 깊이가 n이면 O(n) 공간 복잡도

3️⃣ 선형 자료구조 (Linear Data Structure)

데이터가 일렬로 연결된 구조

3-1. 배열 (Array)
✔ 특징

같은 타입의 데이터

크기 고정

메모리 연속 저장

인덱스를 통한 랜덤 접근

접근 방식

랜덤 접근 → O(1)

순차 접근 → O(n)

장점

빠른 접근

단점

크기 변경 어려움

중간 삽입/삭제 느림 (O(n))

3-2. 연결 리스트 (Linked List)
✔ 특징

노드 단위 저장

비연속 메모리

포인터로 연결

종류

싱글 연결 리스트

이중 연결 리스트

원형 이중 연결 리스트

장점

삽입/삭제 O(1)

단점

탐색 O(n)

랜덤 접근 불가

📌 배열 vs 연결리스트 비교
구분	배열	연결리스트
접근 속도	빠름 O(1)	느림 O(n)
삽입/삭제	느림 O(n)	빠름 O(1)
메모리	연속	비연속

4-2-1. 이진 트리

자식이 최대 2개

종류
종류	특징
정이진트리	0개 또는 2개 자식
완전이진트리	왼쪽부터 채워짐
포화이진트리	모든 레벨이 꽉 참
변질이진트리	한쪽으로 치우침

4-3. 이진 탐색 트리 (BST)

왼쪽 < 루트 < 오른쪽

평균 시간복잡도

탐색: O(log n)

삽입/삭제: O(log n)

단점

한쪽으로 치우치면 O(n)

4-4. AVL 트리

스스로 균형 유지하는 BST

높이 차이 1 이하

회전(Rotation) 사용

항상 O(log n)

4-5. 레드-블랙 트리

색을 이용해 균형 유지

완전한 균형은 아님

삽입/삭제 빠름

Java TreeMap 내부 구현

4-6. 힙 (Heap)

완전 이진 트리 기반

최대 힙

부모 ≥ 자식

최소 힙

부모 ≤ 자식

시간복잡도

삽입: O(log n)

삭제: O(log n)

4-7. 우선순위 큐

우선순위가 높은 데이터 먼저 처리

내부적으로 힙 사용
