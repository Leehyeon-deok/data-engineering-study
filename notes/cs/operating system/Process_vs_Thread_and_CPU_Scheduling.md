운영체제 핵심 개념 정리

(Process · Thread · Memory · IPC · Scheduling)

1. 프로세스(Process)

프로세스란 컴퓨터에서 실행 중인 프로그램을 의미하며,
CPU 스케줄링의 대상이 되는 작업(Task) 과 거의 같은 의미로 사용된다.

프로그램: 디스크에 저장된 실행 파일 (예: chrome.exe)

프로세스: 프로그램이 메모리에 올라가 실행 중인 상태

📌 예시

구글 크롬 실행 파일을 두 번 클릭하면
👉 서로 독립적인 크롬 프로세스 2개가 생성된다.

프로그램이 메모리에 적재되는 과정을 인스턴스화(Instantiation) 라고 하며,
이후 OS의 CPU 스케줄러에 의해 CPU 실행 대상이 된다.

2. 스레드(Thread)

스레드는 프로세스 내부의 작업 흐름(실행 단위) 이다.

프로세스는 1개 이상의 스레드를 가질 수 있다

스레드는 CPU가 실제로 실행하는 최소 단위

📌 핵심

프로세스: 자원 단위

스레드: 실행 단위

3. 정적 라이브러리 vs 동적 라이브러리
3.1 정적 라이브러리 (Static Library)

빌드 시 라이브러리 코드가 실행 파일에 포함됨

실행 파일 크기 증가

장점

외부 의존성 낮음

실행 환경 영향 적음

단점

코드 중복 발생

메모리 효율성 낮음

3.2 동적 라이브러리 (Dynamic Library)

실행 시 필요할 때만 DLL(공유 라이브러리)을 참조

여러 프로세스가 하나의 라이브러리 공유

장점

메모리 효율성 우수

실행 파일 크기 감소

단점

외부 라이브러리 의존도 증가

라이브러리 버전 문제 발생 가능

4. 프로세스 메모리 구조
4.1 Code Segment

프로그램 실행 코드 저장

읽기 전용

4.2 Data Segment

초기화된 전역 변수 / static 변수

0이 아닌 값으로 초기화된 변수

4.3 BSS Segment

초기화되지 않았거나 0으로 초기화된 전역/static 변수

4.4 Heap

동적 메모리 영역

malloc(), free()

런타임 중 크기 변경 가능

4.5 Stack

함수 호출 시 생성

지역 변수, 매개변수, 반환 주소 저장

재귀 호출 시 각 함수 인스턴스가 독립적

5. PCB (Process Control Block)

PCB는 OS가 프로세스를 관리하기 위한 메타데이터 구조체이다.

PCB에 저장되는 정보

프로세스 ID

프로세스 상태

프로그램 카운터

레지스터 정보

메모리 주소 정보

스케줄링 정보

📌 특징

프로세스 생성 시 OS가 PCB 생성

커널 영역에서 관리

일반 사용자는 접근 불가

6. 컨텍스트 스위칭 (Context Switching)

CPU가 다른 프로세스(또는 스레드) 로 전환되며
기존 실행 상태를 PCB에 저장하고 새로운 상태를 복원하는 과정

타임 슬라이스 종료

인터럽트 발생 시 수행

📌 싱글 코어에서도 여러 프로세스가 동시에 실행되는 것처럼 보이는 이유

스레드 컨텍스트 스위칭

스택만 개별

나머지 메모리 공유

프로세스보다 비용이 적음

7. 멀티프로세싱 (Multi-Processing)

여러 프로세스를 동시에 실행

하나의 프로세스 오류가 다른 프로세스에 영향 X

📌 예: 웹 브라우저
(탭 하나 죽어도 전체 브라우저 안 죽음)

장점

안정성, 신뢰성 높음

단점

IPC 비용 큼

8. IPC (Inter-Process Communication)

프로세스 간 데이터 교환 기법

IPC 방식

공유 메모리

파일

소켓

파이프

메시지 큐

익명 파이프

FIFO 구조

단방향

부모–자식 프로세스 간 통신

명명된 파이프

서버–클라이언트 구조

여러 인스턴스 생성 가능

메시지 큐

메시지를 큐 형태로 관리

동기화 문제 해결에 유리

9. 멀티스레딩 (Multi-Threading)

하나의 프로세스 내에서 여러 스레드가 작업을 병렬/동시 처리

장점

자원 공유 → 메모리 효율적

빠른 컨텍스트 스위칭

높은 동시성

단점

한 스레드 오류가 전체 프로세스에 영향

10. 동기화 도구
뮤텍스(Mutex)

공유 자원에 대한 상호 배제

lock / unlock

세마포어(Semaphore)

뮤텍스를 일반화

자원 개수 관리 가능

11. 교착 상태 (Deadlock)

둘 이상의 프로세스가 서로 자원을 기다리며 무한 대기 상태

발생 조건 (4가지)

상호 배제

점유 대기

비선점

환형 대기

해결 방법

예방

회피 (은행원 알고리즘)

탐지 및 회복

12. CPU 스케줄링

CPU를 어떤 프로세스(또는 스레드) 에 할당할지 결정

12.1 비선점형 스케줄링

프로세스가 스스로 CPU 반환

컨텍스트 스위칭 비용 적음

종류

FCFS

SJF

우선순위 스케줄링

12.2 선점형 스케줄링

OS가 강제로 CPU 회수

응답 시간 짧음

종류

Round Robin (RR)

SRF / SRT (선점형 SJF)

다단계 큐 스케줄링

📌 SRT = SRF = 선점형 SJF

🔥 한 줄 요약 (시험용)

프로세스 = 자원 단위

스레드 = 실행 단위

PCB = 프로세스 메타데이터

컨텍스트 스위칭 = 실행 상태 교체

SRT = SRF = 선점형 SJF
