# Java 상속과 다형성 정리

## 1. 상속(Inheritance)의 기본 규칙

### 1) 다중 상속 불가
- 자바에서는 **여러 개의 부모 클래스를 동시에 상속할 수 없다**
- 클래스 상속은 단일 상속만 허용한다

```java
class Child extends Parent1, Parent2 { } // 컴파일 에러
2) 상속 대상에서 제외되는 멤버
private 접근 제한자를 가진 필드와 메서드는 상속 대상에서 제외

부모 클래스와 자식 클래스가 다른 패키지에 있을 경우
default 접근 제한자를 가진 필드와 메서드도 상속 대상에서 제외

2. 메서드 재정의(Method Overriding)
정의
자식 클래스에서 부모 클래스의 메서드를 다시 정의하는 것

재정의 규칙
부모 메서드와 동일한 시그니처를 가져야 한다
(리턴 타입, 메서드 이름, 매개변수 목록)

접근 제한자를 더 강하게 재정의할 수 없다
(public → protected / default / private 불가)

부모 메서드보다 더 넓은 예외를 throws 할 수 없다

부모 메서드 호출
재정의된 부모 클래스의 메서드를 호출해야 할 경우
super 키워드를 사용한다

super.methodName();
3. final 키워드
1) final 클래스
상속할 수 없는 클래스

부모 클래스가 될 수 없으며 자식 클래스를 만들 수 없다

public final class MyClass { }
대표적인 예: String 클래스

2) final 메서드
재정의(오버라이딩)할 수 없는 메서드

메서드 구현을 변경하지 못하도록 제한

public final void method() { }
4. protected 접근 제한자
public과 default의 중간 수준

같은 패키지에서는 default와 동일하게 접근 가능

다른 패키지에서는 자식 클래스만 접근 허용

5. 자동 타입 변환 (업캐스팅)
정의
자식 객체를 부모 타입 변수에 대입하는 것

캐스팅 연산자 없이 자동으로 변환된다

부모타입 변수 = 자식타입;
특징
자식은 부모의 특징과 기능을 상속받기 때문에
부모 타입으로 동일하게 취급될 수 있다

다형성을 구현하기 위해 사용

Animal a = new Dog();
부모 타입으로 선언하면 다양한 자식 객체를 저장할 수 있어
메서드 호출 결과가 달라질 수 있다

6. 강제 타입 변환 (다운캐스팅)
정의
부모 타입을 자식 타입으로 변환하는 것

캐스팅 연산자를 반드시 사용해야 한다

자식타입 변수 = (자식타입) 부모타입;
주의사항
모든 부모 타입 객체를 자식 타입으로 변환할 수는 없다

자식 객체가 부모 타입으로 변환되어 있던 경우에만 가능

잘못된 변환 시 ClassCastException 발생

7. 객체 타입 확인 (instanceof)
강제 타입 변환 전에 객체의 실제 타입을 확인하기 위해 사용

boolean result = 객체 instanceof 타입;
true일 때만 안전하게 다운캐스팅 가능

8. 추상 클래스(Abstract Class)
특징
객체 생성 불가

하나 이상의 추상 메서드를 가질 수 있다

상속받은 자식 클래스는 추상 메서드를 반드시 구현해야 한다

다형성을 구현하기 위한 핵심 구조

abstract class Animal {
    abstract void sound();
}
핵심 요약
상속은 단일 상속만 가능

메서드 재정의는 시그니처와 접근 제한 규칙을 따른다

final 클래스와 메서드는 상속 및 재정의 불가

업캐스팅은 다형성을 위한 핵심 개념

다운캐스팅은 instanceof로 안전성 확인 필요

추상 클래스는 규칙을 강제하고 구현은 자식에게 맡긴다
