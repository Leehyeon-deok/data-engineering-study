Java Platform Independence & Two's Complement 정리


1. Java의 플랫폼 독립성 (Write Once, Run Anywhere)

자바는 “한 번 작성하면 어디서든 실행된다 (WORA)” 라는 슬로건을 가진 언어이다.
이는 플랫폼(운영체제)에 종속되지 않는 프로그램 개발이 가능하다는 의미이다.

1.1 C 언어와 실행 파일의 차이

Windows에서 C 언어로 프로그램을 컴파일하면 .exe 실행 파일이 생성된다.

이 실행 파일은 Windows 운영체제에 맞게 생성된 기계어이므로
Linux나 macOS에서는 실행할 수 없다.

👉 즉, 플랫폼 종속적이다.

1.2 Java 실행 구조

자바에서 Example.java 파일을 컴파일하면 다음과 같은 파일이 생성된다.

Example.class


.class 파일은 완전한 실행 파일이 아님

운영체제에 종속되지 않은 중간 기계어 (바이트 코드)

이 바이트 코드를 실행하려면 JVM(Java Virtual Machine) 이 필요하다.

실행 과정
Java 소스코드 (.java)
 → 컴파일
 → 바이트코드 (.class)
 → JVM
 → 운영체제에 맞는 기계어 실행


즉,

.class 파일 하나만 있으면

각 운영체제에 맞는 JVM만 설치되어 있다면

어느 환경에서든 실행 가능

👉 이것이 자바의 플랫폼 독립성이다.

1.3 Java 성능과 JIT 컴파일러

초기 자바는

인터프리터 방식(한 줄씩 해석)

실행 속도가 느리다는 평가를 받았다.

하지만 현재는 JIT(Just-In-Time) 컴파일러를 사용한다.

JIT 컴파일 방식

자주 반복되는 코드는

매번 기계어로 새로 변환하지 않고

한 번 만든 기계어 코드를 재사용

👉 인터프리터 방식보다 성능이 훨씬 향상

1.4 객체지향 언어의 장점

자바는 객체지향 언어(OOP) 이다.

유지보수가 쉽다

확장성이 좋다

대규모 시스템 개발에 유리

2. 2의 보수 (Two's Complement)
2.1 보수의 개념

보수란 어떤 수를 채워서 특정 값이 되게 만드는 수이다.

10진수에서
→ “3에 대한 N의 보수” = N - 3

10의 보수
→ 더해서 10이 되는 수

2.2 2진수에서의 2의 보수

컴퓨터는 음수를 표현하기 위해 2의 보수 방식을 사용한다.

정의

2의 보수 = 비트 반전 + 1

예시
+5 = 00000101
비트 반전 = 11111010
+ 1
---------
-5 = 11111011

2.3 수식으로 표현한 2의 보수

N비트 시스템에서 어떤 수 X의 2의 보수:

2^n - X

2.4 표현 가능한 값의 범위

n비트일 때 표현 범위:

-2^(n-1)  ~  2^(n-1) - 1


예) 8비트
→ -128 ~ 127

2.5 MSB (Most Significant Bit)

MSB = 최상위 비트

의미:

0 → 양수

1 → 음수

2.6 오버플로우 (Overflow)

오버플로우 발생 조건:

부호가 같은 두 수를 더했는데 결과의 부호가 바뀌는 경우

예)

양수 + 양수 → 음수  (오버플로우)
음수 + 음수 → 양수  (오버플로우)


⚠️ MSB의 carry 발생 여부는 오버플로우 판단 기준이 아니다
👉 부호 변화가 기준

2.7 2의 보수를 사용하는 이유

0이 하나뿐

1의 보수: +0, -0 존재

2의 보수: 0 하나만 존재

하드웨어 구현이 단순

덧셈 회로 하나로

뺄셈까지 처리 가능

2.8 1의 보수와 2의 보수 차이
구분	1의 보수	2의 보수
0의 개수	+0, -0	0 하나
연산	복잡	단순
사용 여부	거의 사용 안 함	실제 컴퓨터 사용
